    Notes on Constructor :

    • Every object has a constructor property and that refers to the function that is used to create that object
    • When we use let obj = {} object literal syntax internally let obj = new Object() happens
    • we have some eg like that we always use literals for some types for eg strings
    • They can also be created as(Built in constructors)
      • let name = new String('manoj')
      • let name = new Boolean()
      • let name = new Number(10)
      • If you print constructor for an object created with fatctory pattern it will show the function used to create

    So if there is a bug in method we have to fix in mulitple places Imagine if there are 100 methods in each object and 100 objects. it would be tedious

    If an object has more than one method then it is said to have a behaviour like car object

    So creating objects with object literal is good if that object has no behaviour

    So what is the solution ?

    Solution : To Use Factory or a constructor function

    Factory Function

    
    function createPerson (name) {
        return  {
            name : name, 
            printName : function(){
                console.log(name)
            }

        }
    }

    const obj = createPerson("manoj")
    manoj.printName()
    Note : if object and key name is same in es6 we can remove nosie code so in that case {name,printName : func}

    Constructor Function 

    
    function Person(name) {
        console.log("this", this)//empty obj 
        this.name = name,
        this.printName = function () {
            console.log(name)
        }
    }

    const obj = new Person('manoj')
    Note :
    • we dont have to return this. it will happen automatically when we use new
    • this is a reference to a object that is executing the current piece of code
    • By default this points to global object , if you are running that piece of code in browser then this points to window object if we are running in node environment then this will refer to global
    • when you call a new operator some things happen under the hood that is create an empty object and will set this to point to that variable which called new operator
    • If we call constructor function without new then if we print this it will show windowobject
    • Instead of accidentally changing global properties and breaking the this will be pointing to empty object when new is called
